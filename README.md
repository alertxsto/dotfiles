# alertxsto/dotfiles

DankMaterialShell (DMS) + Sway dotfiles for Fedora 44.

## What's Inside

| Component | What |
|-----------|------|
| **Sway** | WM config + DMS IPC keybinds, no waybar/swaync/swaylock/swayidle |
| **DMS** | Panel, control center, notifications, OSD, lockscreen, idle |
| **Alacritty** | Terminal with DMS Material You theme |
| **Kitty** | Terminal with DMS Material You theme |
| **flameshot** | Screenshots (PrtSc / Shift+PrtSc) |
| **Auto-theme** | Systemd path watcher syncs DMS colors to Sway client borders |

## Quick Install (Fedora 44)

```bash
sudo dnf install -y git
git clone https://github.com/alertxsto/dotfiles ~/dotfiles
cd ~/dotfiles
./install.sh
```

Then reboot or restart Sway. On first login, run `dms run` once to generate theme files.

## Installation Details

`install.sh` will:
1. Enable DMS COPR + install `dms`, `accountsservice`, `flameshot`
2. Install system matugen templates (`/usr/share/quickshell/dms/`)
3. Symlink configs: `sway/`, `alacritty/`, `kitty/`, `dms/`, `systemd/user/`, `local/bin/`
4. Copy fallback Alacritty theme (Catppuccin Mocha, overwritten by DMS later)
5. Run `alacritty migrate` for TOML config compat
6. Generate Sway client colors from current DMS theme
7. Enable `dms-sway-colors.path` (auto-sync on theme change)
8. Bind `dms.service` to `sway-session.target`

## Auto Theme Sync

When DMS changes theme (light/dark or wallpaper), a systemd path watcher triggers:

1. `dms-sway-colors.path` detects `DankMatugenDark.colors` changes
2. `dms-sway-colors.service` runs `dms-sway-colors` script
3. `swaymsg reload` applies new client colors

## Keybinds

All WM-level binds use `Mod` (Super/Windows key):

| Key | Action |
|-----|--------|
| `Mod+Enter` | Launch terminal (Alacritty) |
| `Mod+Space` | DMS app launcher |
| `Mod+Q` | Close focused window |
| `Mod+Shift+Q` | Kill focused window |
| `Mod+D` | DMS Control Center |
| `Mod+Shift+D` | DMS Network menu |
| `Mod+1-9` | Switch workspace |
| `Mod+Shift+1-9` | Move window to workspace |
| `Mod+arrows/hjkl` | Focus direction |
| `Mod+Shift+arrows/hjkl` | Move window direction |
| `Mod+F` | Toggle fullscreen |
| `Mod+Shift+F` | Toggle floating |
| `Mod+R` | Toggle resize mode |
| `Mod+Shift+C` | Reload config |
| `Mod+Shift+E` | Exit Sway |
| `XF86MonBrightnessUp/Down` | Brightness (DMS OSD) |
| `XF86AudioRaiseLowerVolume` | Volume (DMS OSD) |
| `XF86AudioPlay/Next/Prev` | Media playback (DMS OSD) |
| `PrtSc` | Flameshot GUI |
| `Shift+PrtSc` | Flameshot fullscreen |

## Fresh Fedora Install Checklist

After a clean Fedora 44 install:

```bash
# 1. Update system
sudo dnf upgrade --refresh

# 2. Install git + clone
sudo dnf install -y git
git clone https://github.com/alertxsto/dotfiles ~/dotfiles

# 3. Run installer
cd ~/dotfiles && ./install.sh

# 4. Reboot
systemctl reboot

# 5. On first login, run DMS once
dms run

# 6. Change theme / wallpaper in DMS Control Center (Mod+D)
# Theme colors auto-sync to sway borders and terminals
```

## Development

```bash
# Edit configs in ~/dotfiles/ then:
cd ~/dotfiles
git ac "description of change"
git push

# Or one-liner:
git acp "description"
```

To generate sway colors manually:

```bash
~/.local/bin/dms-sway-colors \
    ~/.local/share/color-schemes/DankMatugenDark.colors \
    ~/.config/sway/dms-colors.conf
swaymsg reload
```

## Notes

- Sway client colors (`dms-colors.conf`) are gitignored (auto-generated)
- Terminal theme files (`dank-*`) are gitignored (auto-generated by DMS matugen)
- Alacritty `dank-theme.toml` is tracked as a fallback; DMS overwrites it
- System matugen templates at `/usr/share/quickshell/dms/matugen/` may be overwritten by DMS updates вЂ” re-copy with `sudo` if needed
